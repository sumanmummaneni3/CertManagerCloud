# CertGuard Cloud

A multi-tenant SaaS platform for SSL/TLS certificate monitoring and management. Track certificate expiry across your entire infrastructure, get alerted before certificates expire, and scan private networks using distributed agents.

---

## Features

### Certificate Monitoring
- **Automated daily scans** of all registered hosts at 2 AM (configurable cron)
- **Expiry alerting** with configurable WARNING (default 30 days) and CRITICAL (default 7 days) thresholds
- **Full certificate metadata** captured per scan: common name, issuer, serial number, expiry, key algorithm, key size, signature algorithm, Subject Alternative Names, chain depth
- **Certificate status tracking**: `VALID` · `EXPIRING` · `EXPIRED` · `UNREACHABLE` · `UNKNOWN`
- **Paginated certificate listing** and expiry-range filtered queries
- **Org-wide dashboard** with aggregate stats (total targets, certs by status, upcoming expirations)

### Target Management
- Register any **domain, IP address, or hostname** with a custom port (default 443)
- Host type (`DOMAIN` / `IP` / `HOSTNAME`) is auto-detected from the host string
- **Enable/disable targets** without deletion to pause monitoring
- Assign targets to **Locations** (AWS, Azure, GCP, colocation, on-prem) for grouping
- **Notification channels per target**: configure email addresses for expiry alerts (SMS, WhatsApp, Slack, Teams, PSA, and ServiceDesk channels are stored for future use)
- Trigger **on-demand scans** without waiting for the scheduled run

### Distributed Agent Scanning
- Deploy lightweight agents on **private networks** unreachable from the cloud
- Agents **self-register** using single-use tokens generated by admins
- Secure agent authentication via **BCrypt-hashed API keys** (`X-Agent-Key` + `X-Agent-Id` headers)
- **mTLS support**: server acts as a Certificate Authority (BouncyCastle) and issues agent client certificates
- Agent **heartbeat** keeps `last_seen_at` updated; agents are listed with status (`PENDING` · `ACTIVE` · `REVOKED` · `EXPIRED`)
- Scan jobs dispatched via a **pull model**: agents poll for pending jobs, claim them, and submit results
- Configurable **CIDR allowlists** restrict which networks each agent is permitted to scan
- **Agent JAR download** and version endpoint served without authentication for easy deployment

### Multi-Tenant Organisation Management
- Full **multi-tenancy**: all data is strictly isolated by organisation — no data leaks between tenants
- Two organisation types:
  - **SINGLE** — standard tenant
  - **MSP** (Managed Service Provider) — can create and manage child client organisations
- **Org profile** management: name, address, contact email, phone
- **Subscription quotas** per org control the maximum number of scanned certificates

### Team & Membership Management
- **Invite members by email** with role assignment
- **Two-step invitation flow**: invite link + OTP verification
- **Org-level roles**: `ADMIN` · `ENGINEER` · `VIEWER`
- Change member roles and revoke access
- MSP admins can manage members across their client organisations

### Role-Based Access Control (RBAC)
| Role | Scope | Capabilities |
|---|---|---|
| `SUPER_ADMIN` | Platform | Manage all org subscriptions and quotas |
| `PLATFORM_ADMIN` | Platform | List all orgs, update quotas, create MSP client orgs |
| `ADMIN` | Org | Manage targets, agents, team members, invitations |
| `ENGINEER` | Org | View targets, certificates, agents, MSP client list |
| `VIEWER` | Org | Read-only access to org data |

### Authentication
- **Google OAuth2 / OIDC** login in production
- **Dev-mode JWT bypass**: issue tokens directly via API without OAuth2 credentials (disabled in production)
- **HMAC-SHA256 JWT** tokens (JJWT 0.12.6), 24-hour expiry, configurable secret

### API & Developer Experience
- **REST API** versioned under `/api/v1/`
- **OpenAPI / Swagger UI** available at `/swagger-ui.html`
- **RFC 9457 ProblemDetail** error responses with consistent structure
- Serves a bundled **React SPA** from the same process — no separate web server needed
- **HTTPS only**: Spring Boot handles TLS directly (TLS 1.2 + 1.3)

### Observability
- **Prometheus metrics** exposed at `/actuator/prometheus`
- **Grafana dashboards** pre-wired to Prometheus
- Structured console logging with configurable log levels

## Getting Started

### Prerequisites

- Docker and Docker Compose
- Java 17 and Maven (to build the JAR)
- A PKCS12 TLS keystore at `./certs/certguard.p12`

### 1. Configure environment

```bash
cp .env.example .env
```

Edit `.env` and set at minimum:

```env
POSTGRES_PASSWORD=your_strong_password
RABBITMQ_DEFAULT_PASS=your_strong_password
JWT_SECRET=replace-with-a-random-64-char-secret   # openssl rand -hex 32
APP_DEV_MODE=true                                  # false requires Google OAuth2 credentials
```

### 2. Build the JAR

```bash
mvn clean package -DskipTests
```

### 3. Start all services

```bash
docker compose up -d
```

### 4. Access the application

| Service | URL |
|---|---|
| Application (HTTPS) | `https://localhost:58244` |
| Swagger UI | `https://localhost:58244/swagger-ui.html` |
| RabbitMQ Management | `http://localhost:15672` |
| Prometheus | `http://localhost:9090` |
| Grafana | `http://localhost:3000` |

### 5. Get a token (dev mode)

```bash
curl -sk -X POST https://localhost:58244/api/v1/auth/dev-token \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@example.com","orgName":"Acme Corp","role":"ADMIN"}'
```

Use the returned JWT as `Authorization: Bearer <token>` on all subsequent requests.

## Configuration Reference

All sensitive values are environment variables. Safe defaults are provided for local development.

| Variable | Default | Description |
|---|---|---|
| `POSTGRES_DB` | `certguard` | Database name |
| `POSTGRES_USER` | `certguard_user` | Database user |
| `POSTGRES_PASSWORD` | — | Database password |
| `RABBITMQ_DEFAULT_USER` | `certguard_rmq` | RabbitMQ username |
| `RABBITMQ_DEFAULT_PASS` | — | RabbitMQ password |
| `RABBITMQ_VHOST` | `/certguard` | RabbitMQ virtual host |
| `JWT_SECRET` | *(dev default)* | HMAC-SHA256 secret, min 64 chars |
| `APP_DEV_MODE` | `true` | `false` enables Google OAuth2 |
| `APP_BASE_URL` | `https://localhost:58244` | Base URL for OAuth2 redirect |
| `GOOGLE_CLIENT_ID` | — | Google OAuth2 client ID |
| `GOOGLE_CLIENT_SECRET` | — | Google OAuth2 client secret |
| `SSL_KEYSTORE_PASSWORD` | `localdevpassword` | PKCS12 keystore password |
| `ALERT_THRESHOLD_WARNING_DAYS` | `30` | Days before expiry for WARNING |
| `ALERT_THRESHOLD_CRITICAL_DAYS` | `7` | Days before expiry for CRITICAL |
| `GRAFANA_PASSWORD` | `changeme` | Grafana admin password |

```
POST   /api/v1/auth/dev-token              Issue JWT (dev mode only)
POST   /api/v1/auth/invite/accept          Accept org invitation

DELETE /api/v1/targets/{id}                Delete target
POST   /api/v1/targets/{id}/scan           Trigger on-demand scan
GET    /api/v1/targets/{id}/scan-status    Poll latest scan status
GET    /api/v1/targets/{id}/notifications  Get notification config
PUT    /api/v1/targets/{id}/notifications  Update notification config

GET    /api/v1/certificates                List certificates (paginated)
```

## Database Migrations

Schema is managed exclusively by Flyway. Hibernate never modifies the schema (`ddl-auto: none`).

| Version | Description |
|---|---|
| V1 | Core tables: organizations, users, subscriptions, targets, certificate_records |
| V2 | Adds `tags` JSONB column to targets |
| V3 | Agent tables: agents, agent_registration_tokens, agent_scan_jobs |
| V4 | `org_invitations` table for member invitation flow |
| V5 | Adds `SUPER_ADMIN` to the `user_role` PostgreSQL enum |
